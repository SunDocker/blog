    -   个人主要工作：
        -   用**单例模式**开发英雄机类（本项目中英雄机实例只允许有一个）

        -   用**策略模式**开发英雄机和敌机的多种子弹射击方式
        -   用**观察者模式**开发炸弹道具
        -   用**数据访问对象模式**开发用户得分排行榜功能
        -   用**模板方法模式**实现三种游戏难度

-   2022.07 ~ 2022.11 哈工大（深圳）ices 实验室 分布式系统项目

    `Java` `gRPC` `RocksDB`

    -   项目名称：doctor 分布式资源调度系统
    -   项目简介：开发调度系统，合理调度数十台服务器的 GPU 资源，为深度学习等计算密集任务提供算力保证，从“一人一张卡”的模式转变为“一次提交多卡服务”的分布式架构。
    -   个人主要工作：
        -   用 Java 语言开发“将镜像拉取至本地”的接口：
            -   在并发场景下，使用`ConcurrentHashMap`和`ReentrantLock`保证并发安全。
            -   使用 **RocksDB** 保存已经拉取的镜像的描述信息。请求此接口时，当一个镜像已经拉取过，则直接返回描述信息，若未拉取，则先拉取并保存描述信息，再返回。
            -   防止并发场景下多次请求导致同一个镜像**重复拉取**的问题。
            -   使用**线程池**异步检查本地镜像数量，若超出一定限制则删除镜像。
            -   使用`RunTime`类封装直接**执行命令行指令的工具类**，用于拉取和删除镜像。
        -   用 Go 语言开发“多服务整合与分发”系统：
            -   为每个服务安装一个`client`，可以与一个统一的`server`建立一个 **gRPC** 长连接，从而实现“服务注册”，每个服务都有其对应的`server id`。
            -   用户的请求会携带`service id`并通过 **HTTP 协议**统一送至`server`，`server`可以据此通过 **gRPC** 长连接将请求分发给对应的`client`。
            -   `client`通过 **gRPC** 长连接收到请求信息后，可以正式向服务发起请求并接收响应信息，再通过 gRPC **长**连接响应给`server`（为每个用户请求分配`request id`以区分不同的请求），再通过`server`的 **HTTP 服务**响应给用户。
        -   使用 Go 语言的 Gin 库和第三方开源库 archiver ，开发将 MinIO 中各类型压缩文件解压的接口

-   2022.09 ~ 2022.11 哈工大（深圳）《编译原理》**课程实验**项目

    `Java`

    -   项目名称：开发简易的编译器
    -   项目简介：用 **Java 语言**编写编译器的**词法分析**器、**语法分析**器、**语义分析**与**中间代码**生成器、**目标代码生成**器，使编译器能够编译特定的语法简单的语言
    -   个人主要工作：
        -   词法分析：确定**编码表**和单词的**正则文法**，用 Java 语言实现从源语言识别出 **token** 的**有穷自动机**。
        -   语法分析：确定识别句子的**上下文无关文法**，使用第三方软件，生成 **LR 分析表**，用 Java 语言编写处理 token 序列的 **LR 分析器。**
        -   语义分析和中间代码生成：确定**语法制导翻译方案**，用 Java 语言实现声明语句、简单赋值和运算语句的翻译。
        -   目标代码生成：确定寄存器分配策略，用 Java 语言实现用中间代码生成 RISCV 汇编语言的程序。
